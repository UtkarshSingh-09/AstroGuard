## ⚠️ ERROR SCENARIOS + HOW TO HANDLE

| Error | When | Handler | Fallback |
|---|---|---|---|
| Groq rate limit (30 RPM) | High traffic | Exponential backoff + queue | Return math result without narration |
| CAMS email delay | CAMS Mailback slow | 5 min timeout | Mode B mock data auto-activates |
| Nifty API down | Behavioral Guard check | Use cached last value | Log warning, skip trigger |
| Twilio WhatsApp fail | Intervention send | Retry 3x | Fall back to browser push |
| MongoDB down | DNA read/write | Redis cache fallback | In-memory session only |
| LLM hallucination | Any narration | Hard-code number check — if LLM output has different numbers than function output → reject | Use template string instead |
| Wrong tax calculation | Tax engine | Unit tests MANDATORY before demo | Pre-computed test case values hardcoded for verification |
| Judge uploads real CAMS | Mode A | Should work — test with real PAN/email before demo | Mode B button clearly visible |
| Demo laptop freezes | Live demo | Pre-recorded backup video of full flow | Sankalp plays video |
